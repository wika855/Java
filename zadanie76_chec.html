Teoria Obiektu Location


Ogólna postać lokalizatora URL:

protocol://host:port/path#fragment?query

Właściwości obiektu location:

Nazwa Znaczenie Dostępność
Hash Część adresu URL znajdująca się za znakiem #. FF, IE3, NN2, OP7
Host Część adresu URL zawierająca nazwę hosta oraz numer portu. FF, IE3, NN2, OP7
Hostname Część adresu URL zawierająca nazwę hosta. FF, IE3, NN2, OP7
Href Pełny adres URL. FF, IE3, NN2, OP7
Pathname Część adresu URL zawierająca ścieżkę dostępu i nazwę pliku. FF, IE3, NN2, OP7
Port Część adresu URL zawierająca numer portu. FF, IE3, NN2, OP7
Protocol Część adresu URL zawierająca nazwę protokołu (np. http, ftp). FF, IE3, NN2, OP7
Search Część adresu URL znajdująca się za znakiem zapytania. FF, IE3, NN2, OP7

Metody obiektu location:

  • assign(url): Wczytuje dokument o adresie wskazanym przez argument url.
  • reload(force): Wymusza ponowne wczytanie bieżącej strony.
  • replace(url): Zastępuje bieżący dokument przez wczytany spod adresu wskazanego przez argument url.
  • toString(): Przekształca zawartość obiektu location w ciąg znaków reprezentujący przechowywany przez niego adres URL.

Różnica między assign a replace:

W przeciwieństwie do metody assign, po zastosowaniu replace bieżąca strona nie zostanie zapisana w historii przeglądanych witryn, co oznacza, że nie będzie można do niej powrócić poprzez przycisk Wstecz bądź history.go(-1).

Przykładowe właściwości i metody obiektu location:

  • window.location.href: Przechowuje pełny adres URL.
  • window.location.hostname: Zawiera wyłącznie domenę.
  • window.location.reload(): Odświeża stronę.
  • window.location.assign(): Przechodzi na podany adres.
  • window.location.replace(): Przechodzi na podany adres i nie tworzy nowego wpisu w historii przeglądania.
